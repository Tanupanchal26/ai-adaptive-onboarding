Algorithms & Logic

Dynamic Gap-to-Path Mapping

# 1. Normalize skills against O*NET taxonomy def normalize(skills): return {s for s in skills if s.lower() in STANDARD_SKILLS} # 2. Compute exact skill gaps gaps = jd_skills - candidate_skills # 3. Find courses covering each gap for course in catalog: if course.skills & gaps: matched.append(course) # 4. Sort: beginner → intermediate → advanced matched.sort(key=lambda c: DIFF_ORDER[c.difficulty]) # 5. Deduplicate by course ID pathway = list({c["id"]: c for c in matched}.values())
  • 1
    Skill Normalization
    Both skill lists mapped to 20-skill O*NET taxonomy — eliminates synonyms ("JS" = "JavaScript")
  • 2
    Set Difference Gap Detection
    gaps = JD_skills − Candidate_skills — O(n) exact matching, zero false positives
  • 3
    Catalog Intersection Search
    Each course's skill set intersected with gaps — multi-gap courses prioritized
  • 4
    Progressive Difficulty Sort
    Beginner → Intermediate → Advanced ensures cognitive load is managed
  • 5
    Deduplication
    One course per gap — no redundant learning, minimum total hours
